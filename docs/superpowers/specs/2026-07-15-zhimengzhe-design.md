## 1. 产品概述

### 1.1 产品名称
**织梦者** — 用文字编织梦境

### 1.2 产品定位
AI 驱动的互动小说可视化阅读器。用户输入故事设定，AI 实时生成场景描述和选项，玩家通过选择推进剧情，获得沉浸式阅读体验。

### 1.3 目标用户
- 喜欢文字冒险、互动小说的玩家
- 想快速体验不同故事设定的创意爱好者
- 对 AI 生成内容感兴趣的早期用户

### 1.4 核心价值
- **零门槛创作**：不需要写剧本，输入一句话设定就能开始玩
- **沉浸式体验**：打字机效果 + 氛围图 + 主题切换，像读小说一样优美
- **无限可能**：每次玩法都不同，AI 实时生成，没有固定剧本

## 3. 技术架构

### 3.1 技术栈
- **框架**: Next.js 14 (App Router)
- **语言**: TypeScript
- **样式**: Tailwind CSS
- **状态管理**: Zustand
- **AI**: DeepSeek API (OpenAI 兼容接口)
- **部署**: Vercel

### 3.2 项目结构

```
织梦者 (Web App)
├── App Router (Next.js 14)
│   ├── /              ← 首页（产品介绍 + 故事设定输入 + 开始游戏）
│   ├── /play          ← 核心阅读器（沉浸式体验）
│   ├── /settings      ← API Key 设置、主题切换、打字机速度
│   └── /premium       ← 付费页面（爱发电链接 + BYOK 解锁）
├── Components
│   ├── StoryReader    ← 核心阅读器（场景显示 + 选项 + 打字机效果）
│   ├── ThemeSwitcher  ← 主题切换（梦境柔光 / 暗夜哥特）
│   ├── OptionButtons  ← 选项按钮组（2-4个选项）
│   └── SceneImage     ← 场景氛围图（预设渐变背景）
├── API Routes
│   └── /api/chat      ← 代理调用 DeepSeek API（流式响应）
└── State (Zustand)
    ├── gameStore      ← 当前游戏状态（场景、历史、选项）
    ├── userStore      ← 用户设置（API Key、主题、体验次数）
    └── themeStore     ← 主题配置
```

### 3.3 数据流

1. 用户在首页输入 **故事设定**（如"你是一名失忆的公主，在梦中寻找记忆"）
2. 点击「开始」→ 进入 `/play`，前端发送设定到 `/api/chat`
3. 后端调用 DeepSeek API，**流式返回** 场景描述 + 2-4 个选项
4. 前端用 **打字机效果** 逐字显示文字，增强沉浸感
5. 用户选择选项 → 将选择发送到 AI，AI 生成下一段场景
6. 循环往复，直到故事结束或用户主动退出

### 3.4 状态持久化
- 所有 AI 对话历史 **存 localStorage**，刷新页面不丢失当前进度
- 体验次数计数 **存 localStorage**，V1 阶段不追求防篡改
- 用户 API Key 和主题偏好 **存 localStorage**

## 4. 页面设计

### 4.1 首页 (`/`)

**功能：** 产品介绍 + 故事设定输入 + 开始入口

**布局：**
- 顶部：产品名称「织梦者」+ slogan「用文字编织梦境」
- 中部：故事设定输入框（textarea，带占位提示）
- 底部：「开始编织梦境」按钮
- 辅助：剩余体验次数提示、设置/关于/支持入口

**交互：**
- 输入框有占位提示示例（如"你是一名失忆的公主，在梦中寻找记忆..."）
- 点击开始按钮 → 验证体验次数 → 跳转 `/play`
- 体验次数为 0 时，点击开始 → 弹出付费引导

### 4.2 阅读器 (`/play`)

**功能：** 核心沉浸式体验

**布局：**
- 顶部：氛围图区域（根据主题显示渐变背景）
- 中部：场景描述文字（打字机效果逐字显示）
- 底部：选项按钮组（2-4个）+ 控制栏（暂停/存档/设置）

**交互：**
- 场景文字打字机效果显示，可调节速度
- 选项按钮 hover 有微动效
- 选择选项后，旧场景上移，新场景从底部打字机显示
- 支持暂停（停止打字机动画）
- 支持存档（手动保存当前进度到 localStorage）

### 4.3 设置页 (`/settings`)

**功能：** 用户配置

**内容：**
- DeepSeek API Key 输入框（密码类型，带显示/隐藏切换）
- 主题切换（梦境柔光 / 暗夜哥特）
- 打字机速度调节（慢/中/快）
- 清空当前游戏进度按钮（需确认）

### 4.4 付费页 (`/premium`)

**功能：** 付费引导 + BYOK 解锁

**内容：**
- 免费体验次数已用完提示
- 爱发电链接（9.9 元/月解锁无限畅玩 + 系统 Key）
- BYOK 入口（填自己的 API Key，永久免费使用阅读器）
- Token 验证输入框（爱发电付费后获取 token，输入后解锁）

## 5. AI Prompt 设计

### 5.1 系统 Prompt

```
你是一位互动小说大师。用户会给出一个故事设定，你需要：

1. 每次只生成一个场景描述（150-300字），营造强烈的画面感和沉浸感
2. 在场景描述后，提供 2-4 个选项供玩家选择
3. 选项要有实际影响力，能真正改变故事走向
4. 保持叙事连贯性，记住之前发生的所有事情
5. 文风要优美、有画面感，像优秀的小说一样

输出格式必须严格如下：

[场景描述]
（空行）
选项：
A. [选项1]
B. [选项2]
C. [选项3]（如果有）
D. [选项4]（如果有）
```

### 5.2 对话流程
- 用户的故事设定作为 system message
- 每轮对话历史都带上，确保 AI 记得上下文
- 选项被用户选择后，将选择内容作为新的 user message 继续对话

### 5.3 错误处理
- AI 输出格式不符合预期（没有选项）→ 前端提示"AI 累了，请重新选择"
- API 超时或失败 → 显示"梦境暂时中断，请稍后重试"
- API Key 无效 → 引导用户到设置页检查 Key

## 6. 变现方案

### 6.1 V1 付费流程

```
用户打开首页
    │
    ├── 已付过费（token 验证） → 无限畅玩
    ├── 填了自己的 API Key → 无限畅玩
    └── 新用户 → 3 次免费体验
         │
         ├── 体验次数 > 0 → 正常玩
         └── 体验次数 = 0 → 弹出付费引导
              │
              ├── 跳转爱发电支付（9.9元/月）
              ├── 输入自己的 API Key 继续玩
              └── 查看设置
```

### 6.2 爱发电验证机制
- 用户付费后，在爱发电填写一个自定义 token
- 回到织梦者，在付费页输入该 token → 存入 localStorage → 解锁无限游玩
- V1 不做自动回调验证，手动 token 录入

### 6.3 定价

| 档位 | 价格 | 说明 |
|------|------|------|
| 免费体验 | 0 元 | 3 次场景对话 |
| 月卡 | 9.9 元/月 | 无限畅玩 + 使用系统 Key |
| BYOK | 0 元 | 填自己的 Key，永久免费使用阅读器 |
| 积分制 (V2) | 待定 | 后续迭代 |

## 10. 技术风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| AI 输出格式不稳定 | 选项解析失败 | 前端做格式容错，正则提取选项 |
| DeepSeek API 响应慢 | 用户体验差 | 流式响应 + 打字机效果掩盖延迟 |
| localStorage 被清除 | 付费状态丢失 | V1 接受此风险，V2 加后端验证 |
| 前端 Key 暴露 | 安全问题 | BYOK 模式下用户自己的 Key，无安全风险 |
| API 成本 | 利润率 | V1 用户自带 Key，零成本；V2 积分制约成本 |
